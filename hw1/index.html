Part 1: Basic Operations

In this section, we apply some basic operations such as brightness, contrast, saturation, gamma and crop. In order to do that more easily, we have modified some classes to make operation easier. First, we created IPixel and FPixel classes similar to Pixel class. IPixel allows us to support negative and > 255 subpixels. FPixel stores float pixels. This is useful, because it not only supports negative values as IPixel, but also has more bits per pixel and allows operations like pow. Moreover, we support conversion between Fpixel, IPixel and Pixel. When we convert from float to int or char, we round the result. We made Image to be a subclass of Buffer<Pixel>. Template class Buffer is a container type of different types of pixels, and supports operations such 2d indexing like (buffer[coord2di(63, 57)]) and iteration through pixels like for(auto &pixel : buffer){};. Because Buffer is a template class, we were able to reuse it for filters. The buffer automatically check if a coordinate is valid. Classes size_2di and coord_2di help to accomplish that.

Brightness, contrast, saturation are basic pixel level operations. The parameter of 1 indicate that image doesn’t changed for all of them. For brightness we just iterate over all pixels and multiply them by the parameter. Because it’s a simple operations, we manipulated pixels as uchar, skipping conversion to int. For contrast, we had to find the grayscale average first. Because multiplication is distributive, we calculated gray scale value for each pixel, while adding to an accumulator. We had to calculate contrast using floating pixels because (Original Pixel - Grayscale Pixel) can be negative. Saturation is handled in similar way, but instead of calculating grayscale average of the image, we use grayscale of the pixel. For gamma change, we simply iterate over all pixel. There, for each pixel, we convert them into float pixel and raise to the power of 1/gamma.

However, because we are performing linear operations in non linear space, numerical errors occur. So, we first need to convert image to linear space, do the transformation, and convert back to original color space. However, because we are using only 8 bits per subpixel, this leads to banding. Because the solution is using simpler method, we decided to also not do conversion to linear space and use the same method as the solution.

For the crop operation, we create a new image and then iterate over all of the new pixels. We find the corresponding coordinate in the old image and if the coordinate is in range of old image size, we copy the pixel, otherwise, we set pixel to black. That way, we can crop image of any size, even if it’s larger than original. Solution crashes during cropping a bigger image, so it’s not shown in comparisons.

Part 2: Quantization and Dithering

In this part, we try three methods to quantize images with a smaller number of bits: linear quantization, random dithering and Floyd-Steinberg dithering. The results are summarized in Figure 2. The number of bits we use to quantize the color value are shown below the images. The first row shows the results by linear quantizing the color value. We can observe that when the number of bits is smaller than 3, there will be very strong color changes. A large area of images are saturated due to the quantization (See the area near the nose of mandrill). However, when the number of bits is larger than 4, we can hardly observe any differences between the quantized images and the original image. The results of random dithering are shown in the second row. Compared with linear quantization, randomly add some noise to the quantization process can alleviate the saturation effect. However, this will bring salt-and-pepper noise which deteriorates the image quality. Among all three methods, the Floyd-Steinberg dithering method can generate images with the most satisfactory visual quality. Even when the number of bits is extremely low (1 bit for each color channel), the color of the images still looks similar to the original one. In Floyd-Steinberg dithering method, the quantization error of each pixel are diffused and accumulated on the neighboring pixels. When implementing the algorithm, we only diffuse the error to the valid pixels, i.e. for the boundary pixels the quantization error that will be diffused out of the images are simply ignored.

The quantitative comparison of the three methods are summarized in Table 1. We use average L1 distance as the error metric. From Table 1, we can see that random dithering will usually lead to higher error compared with the other two methods. Floyd-Steinberg dithering method has very high error when the number of bits is extremely small (1 bits). In other cases, it performs similar to linear quantization in terms of L1 error metric.

Part 3: Basic Convolution and Edge Detection

Filtering is implemented by convolving filter buffer with image. Blur and Sharpen filters first generate integer buffers. For Blur filter, we fist compute the smallest possible element of the kernel. Then we create integer buffer by calculating 2d Gaussian function and dividing it by smallest element. Then, for each pixel in the image, we calculate the sum of product between kernel. Then we normalize the sum by dividing it by sum of the kernel and the image. That way can handle edges and corners. Also, we have to do arithmetics in integers instead of uchars, because pixels can become negative (sharpen) or beyond the range of chars. Sharp filter is implemented is similar way, but the kernel is already known.

The edge detector is similar to blur and edge filters, but we compute convolution for each pixel for both, horizontal and vertical filters together without saving the result. We calculate L2 norm from the grayscale of those two pixels and compare it to threshold.

Part 4: Scale and Shift

We try three filters when scaling or shifting the images. The nearest neighbor filter simply picks out the nearest pixel in the original image and to reconstruct each pixel in the target image. The hat filter and Mitchell filter will do linear and cubic interpolation respectively to alleviate the aliasing and jaggies effect. The kernel function of the two filters are visualized in Figure 4. One interesting observation is that for Mitchell filter, part of the kernel function will be negative, which means that when implementing the algorithm, we should be careful to make sure that all the value of pixels in the target image should be larger than zero. To better compare the difference between different methods, we use checkerboard image as the test example. The results for scale and shift are summarized in Figure 5 and Figure 6 respectively. We enlarge the images so that the details can be better observed. When dealing with image boundaries, we replicate the pixels at the boundaries so that we can obtain valid interpolation for every pixel in the target image. In the case of rescaling the images, we observe that hat filter and Mitchell filter has very similar performance, while nearest neighbor filter will yield distorted checkerboard. In the case of shifting the images, we find that if we shift the image with an integer value, all three methods will lead to the same results. When shifting the image with a real-value, however, we observe that nearest neighbor filter will lead to slight dis-alignment effect, while the results of using hat filter and the Mitchell filter are consistent with each other.
